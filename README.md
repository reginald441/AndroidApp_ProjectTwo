# CS360 Android Mobile App

Android application developed for SNHU CS360.

## Features
- User login
- Database storage
- User interface screens
- Android Studio project

## Technologies
- Java
- Android Studio
- SQLite


## Project Reflection
This project gave me the opportunity to apply the complete mobile application development process from planning and design to implementation and testing. The goal of the application was to create a user-friendly Android app that allows users to log in, manage their information, and interact with the application through a simple and organized interface. The app was designed to address the need for a clean, functional, and easy-to-use mobile experience while following user-centered design principles.

When designing the application, I focused on creating screens that were clear, organized, and easy to navigate. The login screen, user interface, and database features were designed to help users complete tasks with as little confusion as possible. Throughout the project, I learned that understanding user needs is just as important as writing code. A clean layout, logical navigation, and consistent design choices helped improve the overall user experience and made the application easier to use.

My development process involved building the application one feature at a time while testing each part before moving on to the next. I used Java, Android Studio, XML layouts, and SQLite to develop the application. Whenever I encountered build errors or coding issues, I worked through them step by step by reviewing the code, testing changes, and making improvements until everything functioned correctly. This experience taught me the importance of patience, careful debugging, and writing organized code. These are skills I will continue to use in future software development projects.

Testing played an important role throughout the development process. I regularly ran the application in Android Studio to verify that screens loaded correctly, buttons worked as expected, navigation flowed properly, and data was handled correctly. Testing helped identify issues early, making it easier to correct problems before they became larger. It also increased my confidence that the application functioned as intended.

One of the biggest challenges I faced during this project was troubleshooting build and configuration issues while making sure every part of the application worked together correctly. Solving these problems required patience, attention to detail, and a willingness to learn from each obstacle. Looking back at the entire project, I believe my greatest success was building a complete Android application that combined user-centered design, functional features, database integration, and testing into one finished project. This experience strengthened both my technical skills and my confidence as I continue working toward a career in software engineering.
